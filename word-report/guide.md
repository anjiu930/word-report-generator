# A4 型报告编写格式规范指南

> 本文档基于《A4型报告书写要求.docx》整理，用于指导 Word 报告的格式化编写。

---

## ⚠️ 核心禁止规则

| 项目 | 说明 |
|------|------|
| **禁止 Markdown 标记** | 绝对不能使用 `###`、`**`、`--`、`#` 等任何 Markdown 标识符号 |
| **格式控制方式** | 使用纯文本格式，通过字体和段落格式来控制样式 |
| **禁止三级标题**|只接受一级标题或许二级标题，不允许有三级标题 |
| **列表格式** | 列表项使用"数字+点+空格"格式，如 `1.内容`、`2.内容` |

---

## 📐 页面与整体结构

### 保留内容
- **第一章之前的内容**：必须完全保留，不可修改
- **评分表**：必须完全保留，不可修改

---

## 🎯 标题层级规则

### 一级标题（章标题）
- **命名格式**：`第1章 需求分析`、`第2章 系统总体设计` 等
- **字体**：小三号（15pt）
- **加粗**：是
- **段前间距**：0pt
- **段后间距**：4pt
- **分隔**：整个章节之后需要空行分隔

### 二级标题（小节标题）
- **命名格式**：`1.1 设计题目`、`2.1 总体方案的分析与论证` 等
- **字体**：四号（12pt）
- **加粗**：是
- **段前间距**：2pt
- **段后间距**：0pt

### 标题命名注意事项
- 使用参考文档中的原本命名，不要更改
- 确保与参考文档的逻辑相似，不能过于更改

---

## 📝 正文段落规则

### 段落格式
| 项目 | 设置 |
|------|------|
| **字体** | 宋体 |
| **字号** | 五号（10.5pt） |
| **加粗** | 否 |
| **段前间距** | 0 |
| **段后间距** | 0（小段落使用 1磅间距与下一段区分） |
| **首行缩进** | 默认 2 字符缩进 |

### 段落结构要求

#### 普通正文
- 正文段落之间不要分隔行，保持紧凑
- 每段尾部换行后，下一行直接开始下一段
- 使用不同文字长度的段落进行混合书写
- 每段不要以相同的名词开头

### 内容书写规则
- 使用专业术语，保持技术准确性
- 设计方案比对时，仿照参考模板进行比对描述
- 保持段落间的连贯性
- 转换文字表达思路，避免重复开头

---

## 📋 列表规则

### 格式要求
- 使用"数字+点+空格"格式：`1.内容`、`2.内容`
- **尽量少使用列表格式**
- 正文内容多使用段落性文字描述
- 换行进行不同小段的分开

---

## 🖼️ 图片占位符规则
必须预留项目演示图片占位符
### 格式规范
| 项目 | 设置 |
|------|------|
| **格式** | `[图片描述 - 图X]` |
| **对齐** | 居中对齐 |
| **样式** | 斜体 |

### 示例
```
[系统总体框图 - 图1]
[硬件连接示意图 - 图2]
```

## ✅ 必须执行的规则总结

1. **绝对不要使用 Markdown 标记**：`###`、`**`、`--`、`#` 等
2. **不要过度使用空行分隔**：只在每章后使用
3. **不要混合不同的标记方式**：保持格式一致性
4. **不要忘记设置字体大小**：不同层级标题要有明确区分
5. **不要忽略段落间距设置**：通过 `space_before` 和 `space_after` 控制
6. **所有正文文本内容都应该是纯文本**：不包含任何格式标记符号
7. **列表项通过数字加点加空格来表示**
8. **图片占位符通过方括号加描述来表示**
