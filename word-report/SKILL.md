# 📋 核心规则

## 1. 硬件设计内容编写规则
当进行硬件设计时，按以下逻辑编写：
1. **读取项目说明文档**：首先读取用户提供的项目路径中的项目说明文档（如任务书、需求规格说明书等）
2. **自动识别章节位置**：根据文档内容自动确定硬件设计应放在哪一章（可能是第4章、第5章、第6章等）
3. **确定硬件选型**：根据项目要求确定使用的硬件设备
4. **逐个描述硬件**：最少硬件数量需要3到4个：
   - 硬件选择原因和技术特点（1-2段文字描述）
   - 每个被介绍的硬件必须预留原理图/接线图占位图（图X 硬件名称原理图）
5. **硬件连接描述**：整体硬件连接完成后，再添加系统连接图占位符

## 2. 软件设计/代码实现内容编写规则
当进行软件实现章节编写时，按以下逻辑编写：
1. **读取项目源码**：读取用户提供的项目路径中的源代码文件
2. **提取关键代码**：从源码中提取实现核心功能的代码片段
3. **逐段文字说明 + 代码展示**：每个功能点的描述格式为：
   - 先用一段文字说明该功能的实现思路和逻辑
   - 然后展示关键代码（使用代码块格式）
   - 代码后可简要注释说明核心语句的作用
4. **代码选择原则**：
   - 硬件驱动代码：如传感器读取、通信模块配置、执行器控制等
   - 核心算法代码：如数据处理、滤波算法、控制逻辑等
   - 接口代码：如通信协议实现、API调用等
   - 避免展示过长代码，选择最能体现实现思路的片段

## 3.🚨 参考报告处理规则
- **参考报告的作用**：仅用于提取章节标题文字，扫描参考报告，提取一级标题和二级标题中的文字内容
- **template.py 的作用**：固定模板脚本，绝对不能修改其代码结构，只修改中文内容

## 正文段落规则

### 段落格式
| 项目 | 设置 |
|------|------|
| **字体** | 宋体 |
| **字号** | 五号（10.5pt） |
| **加粗** | 否 |
| **段前间距** | 0 |
| **段后间距** | 0 |
| **首行缩进** | 默认 2 字符缩进 |

### 段落结构要求
- 正文小段落之间不要分隔行，保持紧凑
- 每段尾部换行后，下一行直接开始下一小段
- 每小段不要以相同的名词开头

### 内容书写规则
- 保证每小段文字字数不要太相近
- 使用专业术语，保持技术准确性
- 设计方案比对时，仿照参考模板进行比对描述
- 保持段落间的连贯性
- 转换文字表达思路，避免重复开头
- 避免排比句式，不要连续使用"第一...第二...第三..."的列举方式写整段话

## 图片占位符规则

### 放置位置规则
- **只允许在设计与实现章节**：硬件设计，软件设计、系统设计、数据库设计、功能模块开发等
- **测试章节也需要图片**：测试运行章节可以放置运行截图、测试结果图等
- **禁止在非设计章节**：背景介绍，开发环境，开发工具、完成时间、总结体会等章节不放图片

### 引用规则
- 每张图片**必须**在上一段文字中有引用说明
- 引用格式示例："如图X所示"、"图X展示了"、"如图X所示的XXX"
- 图片放在引用它的段落之后

### 格式规范
- **格式**：`图X 图片描述`
- **对齐**：居中对齐
- **样式**：正常（无斜体）

## 参考文献搜索

使用 Chrome DevTools MCP 访问知网搜索文献（知网不需要验证，直接操作即可）。

### 操作流程（严格按照流程来执行）
1. `mcp__chrome-devtools__navigate_page` - 访问 https://www.cnki.net(进入成功后直接采用第二步，不要直接snapshot)
2. `mcp__chrome-devtools__fill` - 在搜索框中输入**简短核心关键词**（如"嵌入式系统 传感器"）
3. `mcp__chrome-devtools__press_key` - 按 Enter 键搜索
4. `mcp__chrome-devtools__take_snapshot` - 获取搜索结果

### 搜索策略
- 知网搜索是**逐字校验**，关键词越多结果越少
- 使用**简短核心关键词**（2-3个词），如：
  - 好：`嵌入式系统`、`传感器`、`STM32`、` 温湿度`
  - 不好：`基于STM32的嵌入式环境监控系统设计`
- 先搜索广泛获取相关文献，再筛选与项目最相关的8篇

### 文献要求
- 8篇近5年内与项目相关的国内文献

### 文献格式规范
```
[1] 作者. 标题[J]. 期刊名称, 年份, 卷(期): 页码.
[2] 作者. 标题[M]. 出版社: 年份.
[3] 作者. 标题[EB/OL]. URL, 年份.
```
