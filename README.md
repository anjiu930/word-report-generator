# Word Report Skill - A4 报告生成助手

> 一款基于 Claude Code 的智能 Word 报告生成工具，专为武汉华夏理工学院学生课程设计全自动写报告打造。

## ✨ 功能特点

- 📄 **自动生成 A4 格式报告**：符合学校课程设计报告格式规范
- 🎯 **模板化生成**：保留封面、目录、评分表等固定格式，只需填充内容
- 📊 **6800 字智能扩展**：自动生成符合字数要求的报告内容
- 🔄 **标题样式支持**：使用 Word 内置 Heading 1/Heading 2 样式
- 📝 **格式规范严格**：严格遵循 A4 型报告书写要求

## 📦 安装配置

### 1. 环境要求

```bash
# Python 版本
Python 3.8+

# 依赖库
pip install python-docx
```

### 2. 目录结构

安装后将 Skill 放置在以下位置：

```
C:\Users\你的用户名\.claude\skills\word-report\
├── SKILL.md              # Skill 配置文件（操作流程）
├── guide.md              # 格式规范指南（核心必读）
├── template.py           # 报告生成脚本模板
└── examples/             # 示例文档
    ├── 课程设计报告模板.docx
    └── 《Android应用开发课程设计》任务书_电信1231-2.doc
```

### 3. 输出目录配置
在skill和脚本中都有相关路径配置，建议手动改为自己倾向的地方。

默认生成目录：
- **Word 文档**：`E:\report\word\`
-**编写的脚本**：`E:\report\py\`

请确保这些目录存在，或修改 `template.py` 中的 `OUTPUT_PATH`。

## 🚀 使用方法

### 工作流程一：直接生成（适合简单项目）

```
用户: "帮我写一份关于 [项目名称] 的报告"

Claude:
1. 读取 guide.md 了解格式规范
2. 读取 template.py 了解代码结构
3. 询问项目基本信息
4. 修改 template.py 中的中文内容
5. 执行脚本生成报告
```

### 工作流程二：项目描述方式（推荐，适合复杂项目）

**优势**：避免扫描大量代码消耗上下文，支持清理后重新生成

```
第一步：生成项目描述（可消耗上下文）
--------------------------------------
用户: "扫描我的项目 [项目路径]，生成项目描述文档"

Claude:
1. 使用 Explore agent 扫描项目结构
2. 提取关键信息：技术栈、功能模块、数据库设计等
3. 生成精简的项目描述文档（保存为 project_description.md）
4. 文档保存到项目根目录


第二步：清理上下文，开启新对话
--------------------------------------
用户: 关闭当前对话，打开新的 Claude Code 会话


第三步：基于描述生成报告（上下文干净）
--------------------------------------
用户: "读取 [项目路径]/project_description.md，生成报告"

Claude:
1. 读取项目描述文档（几百字，上下文消耗小）
2. 直接根据描述修改 template.py
3. 执行脚本生成报告
```

## 📖 核心文件说明

### SKILL.md

**操作流程指南**，包含：
- 触发关键词："编写报告"、"生成报告"、"创建 Word 文档"
- 工作流程和步骤
- 文件输出位置：`E:\report\py\` 和 `E:\report\word\`
- 参考文献搜索流程

### guide.md

**格式规范手册**（核心必读），包含完整的 A4 报告格式规范：

| 规范项 | 要求 |
|--------|------|
| **一级标题** | 小三号（15pt），加粗，段前0pt段后4pt |
| **二级标题** | 四号（12pt），加粗，段前2pt段后0pt |
| **正文** | 宋体五号（10.5pt），首行缩进2字符 |
| **行间距** | 1.25 倍 |
| **禁止** | Markdown 标记、三级标题、过度空行 |

### template.py

报告生成脚本模板，使用方式：

# 1. 修改输出路径
OUTPUT_PATH = r"E:\report\word\项目名称报告.docx"

# 2. 替换各章节的中文内容
add_body_paragraph(doc, "你的项目概述...")
# ... 其他章节内容

# 3. 运行脚本
python template.py
```

## 📋 格式规范速查

### 必须遵守

- ❌ **禁止使用 Markdown**：`###`、`**`、`--`、`#` 等
- ❌ **禁止三级标题**：只能有一级标题和二级标题
- ❌ **禁止过度空行**：只在每章后使用空行分隔
- ✅ **使用数字列表**：`1.内容`、`2.内容`
- ✅ **图片占位符**：`[图片描述 - 图X]`，居中斜体

### 字体格式速查

| 内容 | 字体 | 字号 | 加粗 |
|------|------|------|------|
| 一级标题 | 宋体 | 15pt | 是 |
| 二级标题 | 宋体 | 12pt | 是 |
| 正文 | 宋体 | 10.5pt | 否 |

## 🔧 常见问题

### Q1: 报告字数不够怎么办？

A: 更改字数要求，可适用于不同课程报告要求。

### Q2: 如何添加图片？

A: 使用占位符：
```python
add_image_placeholder(doc, "系统功能模块图", 1)
```
生成后手动替换项目图片最为方便。

### Q3: 如何修改封面信息？

```
请在example目录下的模板报告中填写你的信息，以便于后续每个报告的全自动编写
```

### Q4: 参考文献从哪里来？

A: 默认使用chorme-toolmcp来自动操作浏览器上知网搜寻，这一步搜索文献的流程可手动调整。

### Q5: 为什么生成的报告格式差别过大？
```
请不要过多修改example目录下的报告模板内容，只建议填写课程与个人信息。由于脚本的封面页与评分表采取的是直接复制模版内容，所以更改模版会导致最后的结果出现偏差
```

### 上下文管理技巧（以防止降智）

1. **复杂项目**：先扫描项目生成描述文档，再在新对话中基于描述生成报告
2. **简单项目**：直接告诉 AI 项目信息
3. **多次修改**：保存项目描述文档，每次修改时只需读取文档

### 质量检查

生成报告后检查：
- [ ] 封面信息是否正确
- [ ] 评分表是否在最后一页且完整
- [ ] 图片占位符是否齐全
- [ ] 字数是否接近 6800 字
- [ ] 参考文献 8 篇
- [ ] 格式是否符合 guide.md 规范
